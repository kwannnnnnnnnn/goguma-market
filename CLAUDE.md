# 이 폴더에서 일하는 방식

이 프로젝트는 **개발 연습용** 폴더입니다. 사용자는 비개발자이며, 천천히 배우면서 진행합니다.

## 기술 스택
- **Next.js** — 웹사이트(화면)를 만드는 도구
- **Supabase** — 회원 정보·게시글 같은 데이터를 저장하는 온라인 창고(데이터베이스)
- **Vercel** — 만든 웹사이트를 인터넷에 올려서 누구나 볼 수 있게 해주는 배포 서비스

## 대화 방식
- **쉬운 말로 설명하기**: 개발 용어가 나오면 비개발자도 이해할 수 있게 풀어서 설명합니다.
  예) "API" → "두 프로그램이 서로 정보를 주고받는 통로", "배포" → "인터넷에 올려 공개하기"
- **진행 상황을 자주 알리기**: 지금 무엇을 하고 있는지, 왜 하는지를 작업 중간중간 짧게 말해줍니다.
  사용자가 "지금 뭐 하고 있지?" 하고 궁금해하지 않도록 합니다.

## 작업 방식
- **지시한 것만, 단계적으로**: 사용자가 시킨 일만 합니다. 시키지 않은 기능을 미리 만들거나
  앞서 나가지 않습니다. 한 번에 한 단계씩 진행하고, 다음으로 넘어가기 전에 확인합니다.
- **할 수 있는 건 최대한 직접 처리**: 코드 작성, 파일 수정, 명령어 실행 등 제가 할 수 있는 일은
  최대한 알아서 끝냅니다.
- **사람이 해야 하는 일만 안내**: 제가 못 하는 일(예: 웹사이트에서 직접 버튼 클릭, 비밀번호·키
  입력, 결제, 계정 가입 등)은 **무엇을 / 어디서 / 어떻게** 하면 되는지 그림 그리듯 차근차근
  알려줍니다. 한꺼번에 여러 개를 시키지 않고, 하나씩 안내합니다.

## 마무리
- **작업이 끝나면 바로 확인할 수 있게 열기**: 어떤 작업을 완료하면, 사용자가 결과를 즉시 눈으로
  볼 수 있도록 해당 화면(개발 서버 주소나 배포된 페이지)을 브라우저로 열어줍니다.
  예) 개발 서버를 켜고 `http://localhost:3000` 을 자동으로 열기.
